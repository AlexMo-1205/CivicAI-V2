## Cost model

Routing errors are not equal in harm. We minimize a weighted error cost:

- `local ‚Üí web` (false fallback): cost = **1.0**. Latency / Tavily cost only ‚Äî the web answer is usually still correct.
- `fallback ‚Üí local` (missed fallback): cost = **3.0**. Produces an ungrounded answer from local chunks that don't contain the fact.

## Known ceiling ‚Äî why ~90% is the cap

The reranker scores **single-chunk relevance** between the query and one document chunk. Several `local` multi-doc questions (e.g. `multi-01`, `multi-06`, `multi-12`) cannot be answered from any one chunk ‚Äî the answer is spread across 2‚Äì4 docs ‚Äî so their top reranker score is low (0.50‚Äì0.57). Those scores sit in the same band as the `fallback` near-misses (`fall-04`, `fall-09`, `fall-05`, `fall-06`), where chunks read as topical but the specific fact is absent.

No reranker-score threshold can separate those two groups cleanly. Surpassing the ~90% ceiling would require either:

- a dedicated **answerability classifier** on the top reranked chunk (does this chunk actually contain the answer, beyond being on-topic), or
- fine-tuning the reranker on a `(query, chunk) -> answer-present` signal.

**Both are out of scope for P3c.** This sweep finds the best operating point given the current reranker and corpus.
